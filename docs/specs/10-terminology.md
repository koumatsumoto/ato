# 10. 用語定義: 行動（Action）

## 概念と判断

このアプリで扱うのは、単なる TODO やタスクではない。
思いつき、メモ、将来やるかもしれないこと、実際にやったこと。
それらはすべて、時間の流れの中で連続して存在する。

完了は消去ではなく、跡として残る行為である。

## なぜタスクではないか

タスクは管理と処理のための言葉で、
完了と同時に役目を終える。
このアプリの目的は管理ではなく、
日々を観測し、記録することにある。

## 最終判断: 行動（Action）

行動は、やる前・実行中・完了後のすべてを含められる。

- 思いつきは行動の芽
- やることは行動予定
- 完了は行動の記録

完了しても価値は消えない。
行動は、日々の跡そのものだ。

## 世界観

このアプリはタスクを消すためのものではない。
日々の出来事や思考、実際の行為を
行動として残すためのアプリである。

## ドメインモデル

### 行動 (Action)

日々の跡を残す核となるエンティティ。
思いつき・予定・実行中・完了後のすべての状態を含む。

### タイトル (Title)

行動の名前。1-256 文字。必須。

### メモ (Memo)

行動に付随する補足情報・メモ書き。0-65536 文字。任意。

### ラベル (Label)

行動を分類するためのタグ。0-10 個。1-50 文字/個。
GitHub Issue のラベルとして保存される。

### 状態

- 未完了 (open) - 進行中または未着手の行動
- 完了 (closed) - 達成された行動の記録

## コードとの対応

| ドメイン用語 | コード識別子 | GitHub Issue |
| ------------ | ------------ | ------------ |
| 行動         | Action       | Issue        |
| タイトル     | title        | title        |
| メモ         | memo         | body         |
| ラベル       | labels       | labels       |
| 完了         | closed       | closed       |
| 未完了       | open         | open         |

型名（`Action`, `CreateActionInput` 等）、ファイル名、変数名は全て `action` で統一。
ユーザーに表示されるテキストも全て「行動」で統一する。

ドメインと GitHub API の境界変換は `issue-mapper.ts`（読み取り）と `github-api.ts`（書き込み）で行う。
コード内では常にドメイン用語 (`memo`) を使い、GitHub API の `body` は境界層でのみ使用する。
